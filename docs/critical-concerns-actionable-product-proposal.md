# criticalConcerns 可执行化产品提案

**背景**：`evaluationSummary.criticalConcerns` 返回的是三守护者识别出的具体问题与可执行建议（如「天气存在不确定性，建议调整停留顺序或预留备选方案」）。这些内容可直接用于帮助用户做决策和改行程。

**目标**：从「静态文案展示」升级为「可操作的决策入口」，提高用户采纳率与改行程效率。

---

## 1. 直接用途：显示给用户

### 当前实现
- 在协商结论的「可执行的调整」区块中逐条展示
- 支持点击复制整段文案

### 展示位置建议
| 位置 | 用途 |
|------|------|
| 协商结论「可执行的调整」 | 主入口，已有 |
| 行程卡片 / 日程某天 | 针对该天的提示或小图标说明 |
| 规划工作台侧边栏 | 全局建议摘要 |

---

## 2. 用户可执行的动作（结构化）

以「天气存在不确定性，建议调整停留顺序或预留备选方案」为例，可拆成两条具体动作：

| 建议内容 | 用户可执行的动作 | 产品支持 |
|----------|------------------|----------|
| 调整停留顺序 | 把重要活动放到天气更好的天；把可替代活动放天气差的天 | 交换日程、拖拽重排 |
| 预留备选方案 | 为某天准备 B 计划：若下雨则室内博物馆；若风大则取消观鸟改徒步 | 添加备选活动、Day N 可替换列表 |

---

## 3. 进阶用法：结合产品能力做智能化

### 3.1 展示备选方案
- **示例**：「Day 3 可替换活动：博物馆、温泉、室内景点」
- **产品能力**：从 POI / 知识库获取同区域、同类型备选，按天气/体力筛选
- **入口**：某天的「备选方案」入口，或从 criticalConcerns 点击「查看备选」跳转

### 3.2 重新排程建议
- **示例**：「交换 Day 2 / Day 3」或「把 Day 4 的活动挪到 Day 3」
- **产品能力**：调序、拖拽、一键应用重排
- **入口**：从 criticalConcerns 点击「采纳此建议」→ 跳转改行程界面并预填操作

### 3.3 天气与行程联动
- **示例**：检测到 Day 3 天气不确定性高 → 在 Day 3 日程卡上展示该条 criticalConcern，并给出该日备选活动
- **产品能力**：天气数据 + 行程结构 + criticalConcerns 做智能匹配

### 3.4 一键应用
- **示例**：若系统能生成备选或重排方案，增加「采纳此建议」按钮
- **行为**：跳转到具体改行程界面，并预填对应操作（如交换两天、添加备选）

---

## 4. 实现分阶段建议

### Phase 1：基础（已部分实现）
- [x] 在「可执行的调整」区块中展示 criticalConcerns
- [ ] 优化展示：每条建议配图标/分类（天气、节奏、安全等）
- [ ] 与具体日期/行程段建立关联（若后端能返回 dayIndex 或 segmentId）

### Phase 2：可操作入口
- [ ] 每条建议增加「去改行程」入口，跳转到规划工作台/日程编辑
- [ ] 支持「调整顺序」「添加备选」两类快捷入口，并带上上下文（如 dayIndex）
- [ ] 在日程某天的卡片上展示「该天相关建议」

### Phase 3：智能化
- [ ] 与天气数据联动：按日期展示对应 criticalConcerns
- [ ] 展示该日备选活动列表（依赖 POI/知识库能力）
- [ ] 一键应用：系统生成重排/备选方案，用户一键采纳

## 5. 接口需求（供架构评审）

| 能力 | 当前 | 理想 | 说明 |
|------|------|------|------|
| criticalConcerns 与日期关联 | 无 | dayIndex / segmentIds | 用于在日程某天展示对应建议 |
| 建议类型标签 | 无 | type: WEATHER \| PACE \| SAFETY \| OTHER | 用于分类展示与筛选 |
| 结构化动作 | 无 | suggestedAction: REORDER \| ADD_BACKUP \| ... | 用于生成「采纳」入口 |
| 备选活动推荐 | 无 | alternativeActivities[] | 依赖 POI/知识库，可后续扩展 |

---

## 6. 总结

| 层级 | 做法 | 价值 |
|------|------|------|
| **基础** | 作为说明性文案展示，提醒天气不确定并给出两条方向（调顺序 + 有备选） | 信息透明，降低认知负担 |
| **进阶** | 把两条方向做成可操作入口（改顺序、选备选），结合天气和行程做智能推荐 | 提高采纳率，缩短改行程路径 |
| **智能** | 一键调整 + 备选推荐 + 天气联动 | 从「建议」升级为「决策辅助」 |
