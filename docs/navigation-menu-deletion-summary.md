### **2. 布局结构变更**

**删除前：**
```
┌─────────────────────────────────────────┐
│ [Logo] [Menu] [用户头像]                │ 顶部导航栏
├──────┬──────────────────────────────────┤
│ 左侧 │ [对话历史] │ [对话界面] │ [上下文] │
│ 菜单 │            │            │         │
└──────┴──────────────────────────────────┘
```

**删除后：**
```
┌─────────────────────────────────────────┐
│ [Logo] [用户头像]                        │ 顶部导航栏
├──────────────────────────────────────────┤
│ [对话历史] │ [对话界面] │ [上下文侧边栏] │
│            │            │               │
└──────────────────────────────────────────┘
```

### **3. 功能访问方式**

**删除前：**
- 通过左侧导航菜单访问：工作台、行程库、规划工作台、国家、执行、准备度等

**删除后：**
- **对话界面**：所有功能通过对话界面访问（主要方式）
- **对话历史侧边栏**：Dashboard 页面显示，用于切换会话
- **上下文侧边栏**：Dashboard 页面显示，显示当前行程信息
- **用户头像下拉菜单**：访问设置、个人资料等系统功能
- **移动端底部导航**：保留核心入口（对话、行程、执行）

## 🎯 用户体验变化

### **桌面端**

**删除前：**
- 左侧固定导航菜单（9 个菜单项）
- 需要点击菜单项访问功能
- 侧边栏占用屏幕空间

**删除后：**
- 无左侧导航菜单
- 通过对话界面访问所有功能
- 更多屏幕空间用于主内容
- 符合"对话优先"的产品哲学

### **移动端**

**删除前：**
- 左上角菜单按钮打开侧边栏
- 侧边栏 Sheet 显示导航菜单

**删除后：**
- 使用底部导航栏（`MobileBottomNav`）
- 保留核心入口：对话、行程、执行
- 更符合移动端交互习惯
