## 一、产品原则（必须写进 PRD）

### ① 单决策原则

任何时刻页面只能回答一个问题：

1. **现在情况如何？**
2. **下一步做什么？**

**不是**展示全部信息。

### ② AI 主导原则

系统主动给结论，而不是让用户分析。

| 错误 | 正确 |
|------|------|
| 这里有很多指标，你自己判断 | 我已经判断好了，你确认即可 |

### ③ 渐进揭示原则

```
结论 → 原因 → 数据
```

永远三层。**默认只显示第一层。**

## 二、全新信息架构（核心）

### 当前 vs 建议

| 当前 | 建议 |
|------|------|
| 系统模块驱动 | **决策流程驱动** |

### 新 IA 结构

```
行程页
 ├── 当前状态（唯一核心）
 ├── AI建议（主操作区）
 ├── 今日计划
 ├── 行程时间线
 └── 分析详情（折叠）
```

### 删除项

- ❌ 健康指标条并列
- ❌ 多按钮工具区
- ❌ 权重入口直露

## 三、核心页面原型（文字版）

### 页面顶部（唯一状态区）

```
──────────────────
西峡湾自驾 · Day 2

⚠️ 建议优化后继续
当前评分 78

存在 2 个潜在问题
──────────────────
```

**只有一个状态。用户 3 秒理解。**

### AI 建议区（核心区域）

```
AI发现可以优化的地方：

1️⃣ 今日驾驶时间较长（5.5h）
   可能产生疲劳风险

2️⃣ 下午天气不稳定
   建议调整停留顺序

优化后预计体验提升 23%

[ 一键优化行程 ]
```

**只允许一个主按钮。**

### 今日计划区

```
今日计划
08:30 出发
10:00 景点A
13:00 午餐
16:30 酒店
```

用户熟悉感强。

### 行程时间线

保留现有实现即可。

### 分析详情（折叠）

```
展开：查看分析
```

点击后才出现：安全、节奏、预算、风险、缓冲、专家信息。

## 四、健康度系统重构

### 当前（工程视角）

- 健康度
- 可执行度
- 成本
- 风险
- 缓冲

### 新模型（用户视角）

**只保留一个：**

```
行程评分 92
优秀
```

子维度隐藏于「展开」后：安全 90、节奏 88、预算 95、舒适 91。

**心理负担下降约 80%。**

## 七、实时状态模块重构

### 当前问题

风速、极差、高反、疲劳全部并列 → 用户崩溃。

### 新实时状态

**只显示异常：**

| 正常时 | 异常时 |
|--------|--------|
| 环境状况良好 | ⚠️ 下午可能大风<br>建议提前返回 |

**只显示决策相关。**

---

## 八、按钮哲学（极重要）

### 规则

- 每个页面：**只能有一个主按钮**
- TripNARA 主按钮应该只有三个：

  1. **创建行程**
  2. **一键优化**
  3. **开始导航**

- 其他全部次级。

---

## 九、交互流程升级（黄金流程）

```
进入行程
    ↓
看到状态
    ↓
看到建议
    ↓
点击优化
    ↓
确认
    ↓
完成
```

**整个决策路径：**

- ≤ 5 秒理解
- ≤ 1 次点击完成

这就是 AI 产品优秀体验。

## 十、设计规范原则（可进设计系统）

### TripNARA AI 界面设计原则

| 原则 | 说明 |
|------|------|
| **1️⃣ 单状态原则** | 同一时间只展示一个核心状态 |
| **2️⃣ 单操作原则** | 每屏只允许一个主操作 |
| **3️⃣ 结论优先原则** | 结论永远在原因之前 |
| **4️⃣ 风险显性原则** | 只展示异常，不展示正常 |
| **5️⃣ AI 人格原则** | 文案必须像助手，而不是系统 |

### 文案示例

| ❌ 错误 | ✅ 正确 |
|---------|--------|
| 检测到疲劳指数较高 | 今天可能会有点累，我帮你优化一下 |

## 十三、与现有实现的映射

| 设计原则 | 现有实现 | 待改造方向 |

| 单状态 | 多 Tab、多指标并列 | 收拢为「当前状态」+「AI 建议」 |
