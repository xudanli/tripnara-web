# 行程项生成占位页 - 视觉与交互修改说明

**组件**: `TripGeneratingPlaceholder`  
**路径**: `src/components/trips/TripGeneratingPlaceholder.tsx`  
**场景**: 行程创建后，系统在后台生成行程项（2-5 分钟），此页作为占位展示；或当生成失败时展示错误状态。

> **已实施**：按 TripNARA 视觉原则（Clarity over Charm, Evidence is the aesthetic）完成重构，见下方「五、实施总结」。

---

## 一、原问题（已解决）

### 1. 失败态视觉冲突
- **现象**：标题「生成失败」下方出现被遮挡/截断的「行程功...」类文字，与失败态语义矛盾
- **可能原因**：副标题「行程项生成未能完成，请查看下方详情」在部分视窗下截断或与错误框叠压
- **建议**：失败态时简化副标题，或调整布局避免文字重叠

### 2. 信息层级冗余
- **现象**：「生成失败」出现两次：一次是主标题，一次在红色错误框内（如「生成失败: 候选地点不足...」）
- **建议**：主标题用「生成失败」，错误框内仅展示具体原因，去掉重复的「生成失败:」前缀

### 3. 插画与失败态不匹配
- **现象**：失败态仍保留「进行中」的路径动画（虚线、转圈等），易误导为仍在生成
- **建议**：失败态时使用静态插画或不同的失败态插画，弱化/取消动画

### 4. 按钮文案不统一
- **现象**：部分场景为「重试」，部分为「重新生成」
- **建议**：统一为「重试」或「重新生成」其中一种，并保持全站一致

### 5. 布局与留白
- **现象**：元素堆叠、留白不均，视觉重心不清晰
- **建议**：明确信息层级（主标题 > 错误说明 > 操作按钮），增加适当留白

---

## 二、推荐修改方案（供设计师评审）

| 元素 | 当前 | 建议 |
|------|------|------|
| 主标题（失败态） | 生成失败 | 保持 |
| 副标题（失败态） | 行程项生成未能完成，请查看下方详情 | 简化为「请查看下方错误详情」或移除 |
| 错误框标题 | 完整错误信息（含「生成失败:」） | 仅展示具体原因，如「候选地点不足 (0个)...」 |
| 插画（失败态） | 与进行中相同，含动画 | 静态插画或灰色/简化版本 |
| 主操作按钮 | 重试 | 保持「重试」 |
| 次要操作 | 返回行程列表 | 保持 |

---

## 三、前端实现要点

1. **失败态分支**：`failedMessage` 为真时，整页进入失败态，避免与进行中元素混排
2. **错误文案处理**：若接口返回的 message 已含「生成失败:」，前端可 `replace(/^生成失败[:：]\s*/i, '')` 去掉前缀再展示
3. **插画条件渲染**：失败态时使用 `failure` 插画变体，或对现有 SVG 添加 `className` 禁用动画
4. **响应式**：保证小屏下主标题、错误框、按钮不重叠、不截断

## 五、实施总结（TripNARA 视觉 Designer 输出）

**设计原则落地**：

| 原则 | 实现 |
|------|------|
| Clarity over Charm | 移除装饰性路径/指南针插画，生成中用 Loader2 进度环，失败用 REJECT 圆形图标 |
| Evidence is the aesthetic | 失败态错误块使用 `getGateStatusClasses('REJECT')`，证据式呈现具体原因 |
| Decision is a UI primitive | 失败态映射为 REJECT 裁决，与 GateStatus 系统一致 |
| 四态色彩克制 | 失败用 gate-reject Token，不以大色块渲染情绪 |
| 动效解释状态 | 生成中：Loader2 + 进度条；失败：静态，无动画 |

**Token 与组件**：
- `getGateStatusIcon('REJECT')` / `getGateStatusClasses('REJECT')` 用于失败态
- 中性色：slate 系（800/600/500/200）用于标题、描述、进度
