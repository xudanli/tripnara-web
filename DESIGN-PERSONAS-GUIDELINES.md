# TripNARA「三人格视觉语言总原则」

## 关键词

**安静、理性、陪伴、可信任**

## 整体风格

- 黑白线稿
- 轻手绘
- 圆角、柔和线条
- 少年/青年感，而不是卡通萌娃
- 微笑不是卖萌，而是"礼貌的、温柔的"
- 仅保留 1 个品牌高亮色
  - 用于：✔ 重点提示、✔ 关键信号、✔ 行动建议

---

## 🛡 Abu —— Safety Gatekeeper（安全官）

### ① 人格定位

「有点严肃，但非常可靠的朋友」

- 不会夸张
- 不会可爱
- 但永远站在你这一边

### ② 视觉形象建议

**轮廓**
- 身体偏挺直
- 肩膀略方
- 脸线条简单
- 表情 Calm，而不是紧绷

**眼睛**
- 神情专注
- 眼神略垂直 → 表达判断感

**姿态**
- 常见动作：
  - 手托下巴 → 正在评估
  - 手持路标牌
  - 正在查看等高线/地图
  - 身边放着坡度警示、海拔线、风速 icon

### ③ 标志性元素

| 元素 | 含义 |
|------|------|
| 检查清单 ✔ | 风险管理 |
| 三角⚠ | 风险提醒 |
| 波浪线 🌊 | 涉水 |
| 山形 ⛰ | 海拔 |
| 等高线圈 | DEM |

### ④ 用色

- **主体**：黑白
- **点睛**：🔴 酒红（只用于关键风险标识）

### ⑤ 微表情语言

| 状态 | 表情 |
|------|------|
| 正常 | 平静中性 🙂 |
| 风险轻微 | 眉微蹙 🤔 |
| 严重风险 | 严肃提醒 😐 |

👉 永不夸张  
👉 永远不会惊慌失措  
👉 让用户感到安全、冷静

### ⑥ 文案语气

- 不命令
- 不恐吓
- 有充分理据

**示例：**
> 我注意到这段山路坡度较大，连续三天体能压力会偏高  
> 建议预留缓冲日  
> 你觉得呢？

😌 舒服、可信

## 🎧 Dr.Dre —— Rhythm Architect（节奏官）

### ① 人格定位

「温柔、能共情你身体状态的人类朋友」

- 不是医生
- 但懂你累不累

### ② 视觉设计

**轮廓**
- 动作柔和
- 姿态放松
- 线条流畅

**常见姿势**
- 轻声解释
- 用手比划节奏曲线
- 正在慢慢规划

**旁边元素**
- 曲线（节奏线条）
- 时钟
- 休息椅
- 云朵
- 小音符（象征 rhythm）

### ③ 用色

- 黑白线稿
- 高亮：柔和红/暖橙（非常轻）

👉 感觉是 温柔、人性化，不冒头

### ④ 表情语言

| 状态 | 表情 |
|------|------|
| 默认 | 温柔 🙂 |
| 轻建议 | 微笑轻点头 🙂👍 |
| 担忧轻度疲劳 | 轻皱眉 🤍 |

### ⑤ 文案语气

始终 站在你身体一侧

**示例：**
> 如果你想要旅途中保持轻松  
> 建议每天驾驶控制在 4–5 小时  
> 这样 Day 3 会更舒服一点

像朋友，不是提示框

## 🌊 Neptune —— Plan Repairer（空间修复官）

### ① 人格定位

聪明、灵活、有创造力  
但尊重底线

- 像个理性的艺术家
- 做选择前，他会认真考虑"路线哲学"

### ② 视觉设计

**轮廓**
- 线条流动感
- 身体常带转折
- 手势开放

**常见姿势**
- 手在地图上移动
- 正在替换某一段路线
- 标注关键节点

### ③ 标志性元素

| 元素 | 含义 |
|------|------|
| 地图折线 | RouteDirection |
| 替换箭头 🔁 | 替换 |
| 绿色对勾 | 保留体验 |
| Philo 星标 ✨ | 核心体验未受损 |

### ④ 用色

- 黑白主线
- 高亮：绿色 or 品牌红的柔和版本

表示：
- ✔ 修复成功
- ✔ 不破坏核心体验

### ⑤ 表情语言

| 状态 | 表情 |
|------|------|
| 提方案 | 自信 🙂 |
| 确认可行 | 温柔点头 🙂✨ |
| 担忧破坏体验 | 认真 😐 |

### ⑥ 文案语气

聪明 & 尊重

**示例：**
> 我找到一个替代方案  
> 它可以保留原路线的视野体验  
> 同时减少 40% 驾驶时间  
> 要试试吗？

## 🎨 三人格统一风格守则

| 维度 | 规则 |
|------|------|
| 复杂度 | 极简 |
| 线条 | 稍微粗一点，柔和 |
| 人设年龄 | 青年 |
| 脸型 | 圆润 |
| 情绪 | 平缓 |
| 性别 | 模糊中性 |
| 用色 | 只点睛，不大面积填充 |
| 露面频率 | 不要过多 |
| 互动方式 | 像同伴，而不是吉祥物 |

---

## 🧠 非常重要的一点

他们不是角色化产品卖萌  
他们是「决策透明化的载体」

**目的是：**
- ✔ 让用户知道是谁在判断
- ✔ 同时保持冰山下面是深度系统
